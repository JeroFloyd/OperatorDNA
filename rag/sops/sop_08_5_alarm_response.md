# SOP 08.5 — Alarm Response & Escalation

**Revision:** 5.1
**Effective Date:** 2024-06-01
**Applicable To:** All control room operators

## Purpose

Standard procedure for responding to process alarms to ensure consistent, timely, and documented operator response.

## Alarm Priority Levels

| Level | Color | Description | Response Time |
|-------|-------|-------------|---------------|
| 0 | Green | Normal — no action | — |
| 1 | Yellow | Warning — monitor | Within 30 seconds |
| 2 | Red | Critical — immediate | Within 10 seconds |

## Warning Alarm Response (Level 1)

1. Acknowledge alarm within 5 seconds
2. Identify root cause:
   - Check related process variables
   - Verify sensor status
   - Review recent operator actions
3. Take corrective action per applicable SOP
4. If cause cannot be identified within 60 seconds, escalate to senior operator

## Critical Alarm Response (Level 2)

1. **Acknowledge immediately** — within 2 seconds
2. **Execute pre-determined response:**
   - If pressure alarm: execute SOP 14.2
   - If level alarm: execute SOP 12.4
   - If temperature alarm: execute SOP 15.1
3. **Notify shift supervisor** — within 30 seconds
4. **Document** — log alarm time, response actions, and outcome

## Alarm Flood Protocol

If more than 5 alarms occur within 10 seconds:
1. Acknowledge highest priority alarm first
2. Do not acknowledge lower priority alarms until critical ones are resolved
3. Request support operator if available

## Escalation Criteria

Escalate to senior operator when:
- Operator uncertainty about correct response
- Multiple simultaneous alarms (>3)
- Alarm does not clear after 3 corrective attempts
- Novel alarm pattern not previously encountered
